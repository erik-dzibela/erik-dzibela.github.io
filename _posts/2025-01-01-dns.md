**Domain Name System (DNS)** is a hierarchical and distributed naming system used to make forward and reverse DNS lookups. Or, more simply, it is used to resolve a name to an IP address, and vice versa.

This is what allows us to use simple names like r00t26.com when accessing websites and web apps, instead of using complex IP addresses.

DNS operates on **port 53**, mainly using **UDP**. DNS uses TCP for zone transfers, and UDP for DNS queries (e.g. forward and reverse lookups). UDP can be used to exchange relatively small bits of information, whereas **TCP must be used to exchange information that is larger than 512 bytes**. TCP is chosen for zone transfers because it provides features like error-checking, reliability, and the ability to handle larger data payloads, which are important for ensuring accurate and complete synchronization of DNS data between servers.

When resolving domain names and IP addresses, DNS uses two types of queries or "lookups":

- **Forward DNS lookup** — resolves a name to an IP address.
- **Reverse DNS lookup** — resolves an IP address to a name.

## Domain Hierarchy

The domain hierarchy is employed to arrange the components of a domain name based on their level of significance.

A **TLD** (Top-Level Domain) is the right part of a domain name — for example in r00t26.com, **.com** is a TLD.

There are two types of TLDs:

- **gTLD** — Generic Top-Level Domain (e.g. `.com`, `.gov`, `.edu`, `.org`)
- **ccTLD** — Country Code Top-Level Domain (e.g. `.ca` for Canada, `.sk` for Slovakia, `.co.uk` for the UK)

A TLD helps classify a website's purpose or geographical origin. For instance, r00t26.com is **not the same** as r00t26.org.

**Second-Level Domain (SLD)** is the customizable portion — like **r00t26** in r00t26.com.

**Subdomains** sit to the left of the SLD, e.g., **contact.r00t26.com**. These are often used to separate site sections. Note: **www** is technically a subdomain too!

## How DNS Works

1. **User Query** — User enters a domain (e.g., r00t26.com).
2. **Local Cache** — System checks if the IP is already cached.
3. **Recursive DNS Resolver** — Queries ISP or public resolver.
4. **Root DNS Servers** — Resolver contacts root servers.
5. **TLD Servers** — Directed to `.com`/`.org` DNS servers.
6. **Authoritative DNS Servers** — TLDs direct to the domain's DNS server.
7. **Querying Authoritative Server** — Domain records are requested.
8. **DNS Records** — An A record is returned with the IP address.
9. **Caching** — Info is cached for future lookups.
10. **Response to User** — IP is returned and browser connects.

## DNS Record Types

DNS records provide instructions on how a domain functions. Key types include:

| Record | Description |
|--------|-------------|
| **A** | IPv4 address mapping |
| **AAAA** | IPv6 address mapping |
| **CNAME** | Alias of another domain |
| **MX** | Mail server address |
| **TXT** | Text data (e.g., SPF, DKIM) |
| **SRV** | Service info (e.g., VoIP) |
| **NS** | Authoritative name servers |
| **PTR** | Reverse DNS (IP to name) |
| **SOA** | Start of authority (admin info) |
| **CAA** | Certificate authority restrictions |

## Summary

DNS is the backbone of the internet's naming system. It translates human-friendly domain names into IP addresses, allowing devices to connect efficiently. Understanding its hierarchy, record types, and resolution flow is essential for security pros, developers, and sysadmins alike.

As you continue exploring DNS, you'll uncover more intricacies that show just how critical it is to the web's functioning.
